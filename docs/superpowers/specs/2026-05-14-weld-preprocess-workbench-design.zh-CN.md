# 焊前工艺设定工作台设计

## 目标

建设一个焊前工艺设定模块，使一个工件 STEP 文件对应一份焊接工艺计划。目标用户是焊接工程师，而不是机械设计或结构工程师，因此产品不应做成传统 CAD 软件。它应聚焦于低学习成本的焊缝标注、焊接阶段定义、焊接顺序设定，以及激光焊枪姿态预览。

系统不应依赖自动焊缝识别。一个订单中的工件通常只需要标注一次，而即使同属 manifold，不同工件的水嘴数量、左右端结构和异构件也可能差异很大。第一版应优先保证人工标注可靠、可解释，并提供足够强的视觉反馈。

## 核心产品模型

原始 STEP 是几何真源。焊接阶段是工艺对象，不要求用户先把 STEP 拆分成多个文件。

```text
工件 STEP
  -> 几何显示与拾取元数据
  -> 焊接工艺计划 Weld Plan
      -> 焊接阶段 Weld Stage
          -> 焊缝组 Weld Seam Group
              -> 焊缝段 Weld Seam Segment
              -> 焊接顺序 Weld Order
              -> 激光姿态定义 Laser Pose Definition
```

阶段可以表示装夹、工位、可达性、工艺意图或焊接策略。例如一个 manifold 可以有中部水嘴阶段、左端旋转焊阶段、右端异构端阶段，但这些阶段不必对应拆分后的 STEP 文件。

## 技术方向

采用 Web 工作台加后端 STEP 预处理的架构。

前端：

- React 和 TypeScript。
- Three.js 或 React Three Fiber 负责 3D 渲染。
- Zustand 或同类轻量状态管理，维护当前标注状态。
- TanStack Query 或同类方案，管理几何处理任务、保存和加载。
- 自定义受限 3D 交互层，不复刻 CAD 式复杂操作。

后端：

- Python FastAPI 服务。
- 通过 OpenCascade 生态的 OCP/CadQuery 读取 STEP、网格化、提取拓扑、分类曲线并生成拾取元数据。
- 将显示用几何保存为 GLB 或同类轻量 Web 格式。
- 将焊接工艺计划与 STEP 分离保存，使用结构化 JSON 或数据库记录。

性能策略：

- STEP 只在后端解析。
- 前端加载简化后的显示网格。
- 精确拾取依赖边、曲线、面引用等元数据。
- 高亮和焊缝标记使用独立 overlay 几何，不直接改动大模型材质。
- 通过 STEP hash 缓存派生几何数据。

## 标注交互

标注以人工为主。系统帮助用户拾取几何，但不替用户判断哪些位置是焊缝。

目标形状模式：

- Circle：圆形边和圆弧边。这里包括非闭合圆弧，不只是真正的整圆。
- Rectangle：矩形边框或四边闭合轮廓。
- Edge：自由边逐段选择。

Circle 模式下，系统应识别鼠标附近的圆形曲线，并提供半径、圆心、法向、弧长、闭合状态等信息。用户靠近圆边或圆弧时，只高亮当前候选。若多个圆弧段共同构成一个实际焊缝，用户可以继续选择相邻圆弧，将它们组合成同一焊缝段。

批量选择应保持克制：

- 允许同直径选择。
- 第一版不做同方向、等间距或相似结构推断。

这样可以让系统行为足够可解释。工程师仍然负责判断几何是否为焊缝。

## 视觉交互

视觉清晰度是核心产品能力。

建议的视觉状态：

- 未标注几何：中性灰色。
- 鼠标悬停候选：蓝色。
- 当前正在编辑的焊缝：黄色。
- 已确认焊缝：绿色。
- 当前阶段或当前焊缝组：紫色或橙色强调。
- 冲突或缺失信息：红色。

焊缝标记应使用独立 overlay：

- 沿选中路径生成略微抬起的线或细管。
- 沿焊缝方向显示方向箭头。
- 显示阶段、焊缝组和序号标签。
- 当前阶段焊缝保持突出，其他阶段半透明弱化。
- 3D 视图和焊缝列表双向高亮。

3D 视图主要负责空间确认。顺序管理和批量管理应主要在侧边面板与列表中完成。

## 焊接阶段与顺序

用户创建阶段，并将焊缝组分配到阶段中。阶段可以代表一次装夹、一个工位、一个机器人站位、一次旋转工艺，或一种控制热输入的焊接策略。

顺序设定应以列表为主：

- 拖拽调整焊缝或焊缝组顺序。
- 手动编辑序号。
- 批量重编号。
- 对选中焊缝反转焊接方向。

对于 manifold 中重复水嘴，后续可以支持简单顺序模板。但第一版应以手动排序和清晰的 3D 序号标签为可靠基线。

## 激光姿态定义

激光角度不应只是几个静态数字，而应建模为沿焊缝路径变化的姿态场。

工程师需要定义：

- 焊缝路径。
- 参考法向方向。
- 焊接行进方向。
- 少量工艺语义下的角度与偏置参数。

系统根据这些定义，沿整条焊缝采样计算焊枪姿态，并预览焊枪在焊接过程中的运动。

参考法向来源：

- 选中的相邻面。
- 另一个相邻面。
- 两个面的夹角平分方向。
- 手动方向。

用户必须能够显式翻转法向方向。

可编辑工艺参数：

- 工作角或入射角。
- 行进角。
- 侧向偏置。
- 焦点高度或焦点偏移。
- 如工艺需要，可加入光斑或离焦参数。
- 行进方向正向或反向。

主界面不暴露原始矩阵、欧拉角或机器人品牌相关姿态字段。

## 焊枪姿态预览

姿态预览必须展示焊接过程，尤其是圆形或曲线焊缝。

预览模式：

- 当前采样点姿态：通过进度条或拖动控件查看某一点的焊枪和激光束。
- 姿态 ghost：沿焊缝显示多个半透明焊枪姿态。
- 连续动画：焊枪沿焊缝运动，帮助用户判断姿态是否合理。

预览应包含：

- 带方向的焊缝路径。
- 被选中的参考法向箭头。
- 打到焊缝上的激光束或光锥。
- 可选的局部切线、法向和侧向坐标轴。
- 对姿态不连续、缺失法向、法向翻转或采样点不可解的位置做红色标记。

对于圆形焊缝，焊枪姿态必须沿路径旋转。单个静态箭头不足以表达整个焊接过程。

## 数据形态

焊接工艺定义应以可基于几何重放的形式保存。

激光姿态定义示例：

```json
{
  "referenceNormal": {
    "type": "face",
    "faceId": "face_123",
    "flipped": false
  },
  "travelDirection": "forward",
  "workAngleDeg": 35,
  "travelAngleDeg": 10,
  "lateralOffsetMm": 0,
  "focusOffsetMm": 2,
  "sampleCount": 32
}
```

每条焊缝应同时保存精确几何引用和备用采样路径：

- 可用时保存 STEP 派生的边或面引用。
- 对圆弧保存曲线类型与几何参数。
- 保存备用 3D polyline 采样点，用于重放和视觉连续性。
- 保存阶段、焊缝组、顺序和姿态引用。

这样即使后续预处理导致拓扑 ID 变化，工艺计划仍有机会通过几何参数和采样路径恢复。

## 第一版非目标

- 完整自动焊缝识别。
- 基于 LLM 的识别。
- 超出同直径辅助选择的复杂模式识别。
- 完整 CAD 编辑。
- 要求用户按阶段强制拆 STEP。
- 在主工艺模型中直接绑定具体机器人品牌。
- 向焊接工程师暴露机器人矩阵或欧拉角。

## 未决问题

- 最终会由哪类机器人或离线编程系统消费焊接计划。
- 焦点高度和离焦参数是否进入第一版姿态 UI。
- 阶段中需要表达多少夹具、装夹或工位元数据。
- 矩形焊缝是否常见到值得在第一版投入与圆形焊缝同等的交互打磨。

## 建议下一步

为第一版原型制定实现计划：

1. STEP 预处理为 GLB 加边/曲线元数据。
2. Web 3D 查看器支持 Circle、Rectangle、Edge 三种选择模式。
3. 焊缝 overlay、阶段和焊缝组列表管理。
4. 基于参考法向的激光姿态定义，以及 ghost/动画姿态预览。
5. 保存与工件绑定的 JSON 焊接计划。
